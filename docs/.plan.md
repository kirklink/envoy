# Plan: Recall Evaluation Harness

## Goal
A semi-automated harness for tuning `RecallConfig` weights. Change weights → run
`dart run bin/eval.dart` → see what improved / regressed vs the previous run.
Results accumulate in `eval/results.jsonl` so trends are visible over time.

---

## What gets built

### New files (all inside `envoy/souvenir/`)

```
lib/src/eval/
  types.dart      ← EvalScenario, EvalQuery, QueryResult, ScenarioResult, RunSummary
  runner.dart     ← EvalRunner: seeds store, embeds, runs recall, scores
  scenarios.dart  ← 8 built-in scenarios (adapts QualityTestEmbeddings + _seedMemories)
  report.dart     ← console table, markdown, JSONL serialisation

bin/eval.dart     ← CLI entry point

eval/
  .gitkeep        ← keeps the dir in git
  results.jsonl   ← append-only, one JSON object per run (gitignored, created on 1st run)
  latest.md       ← human-readable report, overwritten each run
```

`lib/eval.dart` — thin barrel (public entry point for tests if needed)

## Types (types.dart)

```dart
class EvalQuery {
  final String query;
  final String expectedTopFragment; // substring expected in #1 result content
  final String description;
}

class EvalScenario {
  final String name;
  final String description;
  final Future<void> Function(MemoryStore, EmbeddingProvider?) setup;
  final List<EvalQuery> queries;
}

class QueryResult {
  final EvalQuery query;
  final bool pass;          // expectedTopFragment found at rank 1
  final int rank;           // 1-based rank of expected memory (0 = not found)
  final double reciprocalRank; // 1/rank, 0 if not found
  final List<ScoredRecall> actual; // full ranked list with score breakdowns
}

class ScenarioResult {
  final EvalScenario scenario;
  final List<QueryResult> queryResults;
  double get passRate;
  double get mrr;
}

class RunSummary {
  final DateTime timestamp;
  final RecallConfig config;
  final String embeddingMode;  // "fake" | "ollama:model-name"
  final List<ScenarioResult> scenarioResults;
  double get overallMrr;
  double get overallPassRate;
  int get totalQueries;
  int get passedQueries;
}
```

## Scenarios (8 built-in)

Adapted from `recall_quality_test.dart` — reuses `QualityTestEmbeddings` and
`_seedMemories`. Each scenario is a standalone struct: fresh store, setup fn, queries.

| # | Name | Tests |
|---|------|-------|
| 1 | `semantic_bridge` | "favourite animal" → rabbits via vector (no FTS match) |
| 2 | `fts_direct` | "REST API" → task goal via keyword match |
| 3 | `entity_expansion` | "Alice" → rabbit memory via entity graph (1-hop) |
| 4 | `multi_signal` | Dart query → Dart memory ranks above PostgreSQL via multi-signal |
| 5 | `component_weights` | Durable fact outranks environmental for long-term query |
| 6 | `temporal_decay` | Recent memory outranks identical old memory |
| 7 | `relevance_silence` | "quantum entanglement" returns nothing (noise rejection) |
| 8 | `conversation_pipeline` | Multi-step seed simulating consolidated conversation turns |

## EvalRunner (runner.dart)

```dart
class EvalRunner {
  final RecallConfig config;
  final EmbeddingProvider? embeddings;

  Future<RunSummary> runAll(List<EvalScenario> scenarios) async {
    // For each scenario:
    //   1. Fresh InMemoryMemoryStore
    //   2. Call scenario.setup(store, embeddings) — seeds memories + entities
    //   3. For each query: run UnifiedRecall, find rank of expected fragment
    //   4. Collect QueryResult
    // Return RunSummary
  }
}
```

Key: each scenario's setup function handles embedding. With real Ollama, it just
passes the provider; with fake, it uses QualityTestEmbeddings (same as tests).

## CLI (bin/eval.dart)

```
dart run bin/eval.dart [options]

Options:
  --fts-weight <n>      FTS signal weight (default: 1.0)
  --vec-weight <n>      Vector signal weight (default: 1.5)
  --entity-weight <n>   Entity graph weight (default: 0.8)
  --threshold <n>       Relevance threshold (default: 0.05)
  --ollama              Use Ollama embeddings (requires local Ollama)
  --ollama-model <m>    Ollama model (default: all-minilm)
  --ollama-host <url>   Ollama host (default: http://localhost:11434)
  --compare             Show delta vs previous run only
```

## Results format (eval/results.jsonl)

One JSON object per line (append-only):
```json
{
  "timestamp": "2026-02-24T14:30:00Z",
  "config": {"ftsWeight": 1.0, "vectorWeight": 1.5, "entityWeight": 0.8, "threshold": 0.05},
  "embeddingMode": "fake",
  "overallMrr": 0.96,
  "overallPassRate": 0.90,
  "totalQueries": 20,
  "passedQueries": 18,
  "scenarios": [
    {
      "name": "semantic_bridge",
      "queries": 3, "passed": 3, "mrr": 1.00,
      "queryDetails": [
        {"query": "favourite animal", "rank": 1, "pass": true, "topContent": "User thinks rabbits..."}
      ]
    }
  ]
}
```

`eval/results.jsonl` is gitignored (it grows per run, belongs to the developer's
local tuning session). `eval/latest.md` is also gitignored for the same reason.
