# Envoy Status — 2026-03-10

## Package Structure

Three packages in nested layout:

- **`envoy/`** — Core agent framework (`EnvoyAgent`, `EnvoyContext`, `Tool`, permission tiers)
- **`envoy_tools/`** — Seed tools + persistence layer (Stanza-backed tool registry, session history)
- **`souvenir/`** — Persistent memory engine (unified recall, component-based consolidation)

## Phase Progress

## Open Design Questions (from agent_plan.md)

1. **Helper agents** — How do primary agents delegate to specialized sub-agents?
2. **Provider abstraction** — How thin is the LLM client interface? (Anthropic-only for now)
3. **Context window strategy** — Current pruning drops entire message pairs; better approach is distillation
4. **Tool versioning** — Immutable tool versions needed if updated
5. **Streaming UX** — Loop needs streaming support (deferred to Phase 5)
6. **Network tool allowlists** — Phase 2 deferred; needs addressing before production
7. **Dynamic tool I/O** — Currently JSON via args; may hit OS arg-length limits (migrate to stdin)

## Envoy Dashboard Status

Full-stack agent monitoring UI at `envoy_dashboard/` — ~754 lines total.

**Structure:** `client/` (Trellis SPA) + `server/` (Swoop backend)

**What works:**
- Server runs agents and streams events via SSE
- Trellis UI renders with dark mode, routing, tool call tables
- Dashboard, Chat, and Logs pages functional

**Gaps:**
- Static file serving not implemented (client not served by server)
- No `analysis_options.yaml` in client or server
- No README.md or CLAUDE.md
- CORS hardcoded to `localhost:8084`
- No production build process
